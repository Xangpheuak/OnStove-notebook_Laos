# OnStove Laos – Input Data

This folder contains all input data required to run the **OnStove model for Lao PDR (Laos)**. OnStove is a geospatial cost-benefit clean cooking tool that calculates the net-benefits of different cooking solutions for each raster grid cell of a given study area. The tool accounts for four benefits (reduced morbidity, mortality, greenhouse gas emissions, and time saved) and three costs (capital, fuel, and operation & maintenance).

> **Reference model:** [OnStove – KTH dESA](https://github.com/Open-Source-Spatial-Clean-Cooking-Tool/OnStove)  
> **Coordinate Reference System:** UTM Zone 48N (EPSG: 32648)

## Data Description

### 1. Scenario Specification Files (CSV)

#### `tech_specs_lao.csv`
Contains technical parameters for all cooking technologies modelled in the Laos study. Each row represents one parameter for one fuel/technology.

| Column | Description |
|--------|-------------|
| `Fuel` | Name of the cooking technology |
| `Param` | Parameter name |
| `Value` | Parameter value |
| `Type` | Data type (`float`, `str`) |
| `Unit` | Unit of measurement |
| `Reference` | Source reference |

**Technologies included:**

| Technology | Description |
|------------|-------------|
| `Traditional_Biomass_Tripod` | Traditional open-fire tripod stove (dominant rural baseline) |
| `Traditional_Biomass_TaoCement` | Traditional cement Tao stove |
| `Traditional_Biomass_TaoDam` | Traditional black Tao stove (Tao Dam) |
| `Traditional_Biomass_TaoPayat` | Improved traditional Tao Payat stove |
| `Biomass_ICS_WS-3` | Improved Cookstove – wood-saving type 3 (WS-3) |
| `Biomass_ICS_PPT-3` | Improved Cookstove – pot/pressure type 3 (PPT-3) |
| `LPG` | Liquefied Petroleum Gas stove |
| `Electricity` | Electric cooking (coil/infrared) |
| `Biogas` | Household biogas digester system |

**Key parameters per technology:** efficiency, fuel cost, investment cost, O&M cost, technology lifetime, energy content, cooking time, fuel collection time, CO₂ intensity, CH₄ intensity, N₂O intensity, BC intensity, OC intensity, PM2.5 intensity, current adoption share (urban/rural).

**Primary sources:** GERES / SNV Lao PDR ICS Program, IPCC (2006) Guidelines, GAINS Model (IIASA), Bond et al. (2013), Clean Cooking Alliance (CCA), EDL Residential Tariff 2024, Ministry of Industry & Commerce Lao PDR, World Bank / Local Market Surveys.

#### `soc_specs_lao.csv`
Contains socioeconomic scenario parameters and health burden data for Laos.

| Column | Description |
|--------|-------------|
| `Param` | Parameter name |
| `Value` | Parameter value |
| `data_type` | Data type (`float`, `string`, `int`) |
| `Reference` | Source reference |

**Key parameters:**

| Category | Parameters |
|----------|------------|
| **Demographics** | Population (2025): ~7,873,000; Urban share: 38%; Rural/Urban household size: 5.8 / 5.5 persons |
| **Electrification** | National electrification rate: 95%; Rural: 90%; Urban: 99% |
| **Health burden (mortality per 100k)** | COPD: 15.2; IHD: 85.5; Lung cancer: 12.4; ALRI: 45.2; Stroke: 95.4 |
| **Health burden (morbidity per 100k)** | COPD: 450.5; IHD: 520; Lung cancer: 10.2; ALRI: 3,500; Stroke: 410.5 |
| **Economics** | Minimum wage: 126 USD/month; VSL: 134,765 USD; Discount rate: 15%; Carbon cost: 51 USD/ton |
| **Model weights** | Health: 1; Time: 1; Costs: 1; Environment: 0; Spillovers: 0 |
| **Biomass** | Non-renewable biomass fraction (fnrb): 0.15 |

**Primary sources:** Lao Social Indicator Survey (LSIS III) 2023, UN World Population Prospects 2024, IHME Global Burden of Disease 2019, World Bank WDI, Lao Ministry of Energy and Mines, UNFCCC CDM TOOL33, WHO CHOICE.

### 2. GIS Data (`gis_data/`)

All raw geospatial layers are stored here. The notebook (`OnStove_notebook_Laos.ipynb`) reads these files, reprojects and aligns them, then saves the processed versions into `Processed_Data_Laos/`.

#### Administrative
| File | Format | Description | Source |
|------|--------|-------------|--------|
| `Laos_boundary.geojson` | Vector (GeoJSON) | National boundary of Lao PDR used as the study area mask | Lao Statistics Bureau / GADM |

#### Base
| File | Format | Description |
|------|--------|-------------|
| Base raster | GeoTIFF (`.tif`) | Reference base layer used for spatial alignment of all other rasters |

#### Biomass
| File | Format | Description | Source |
|------|--------|-------------|--------|
| `Forest/Forest.tif` | Raster (GeoTIFF) | Forest cover — used to estimate biomass availability for cooking | Global Forest Watch / Hansen et al. |
| `Friction/Friction.tif` | Raster (GeoTIFF) | Travel friction surface — used to estimate fuel collection time | Weiss et al. (2018) / Malaria Atlas Project |

#### Demographics
| File | Format | Description | Source |
|------|--------|-------------|--------|
| `population/Population.tif` | Raster (GeoTIFF) | Population density (persons per grid cell) | WorldPop / Facebook HRSL |

#### Electricity
| File | Format | Description | Source |
|------|--------|-------------|--------|
| `MV_lines/MV_lines.geojson` | Vector (GeoJSON) | Medium-voltage electricity transmission lines | Lao Electricity (EDL) / OpenStreetMap |
| `Night_time_lights/Night_time_lights.tif` | Raster (GeoTIFF) | Night-time light intensity — proxy for electrification status | VIIRS/NPP Nighttime Lights (NASA/NOAA) |

#### Livestock
Livestock density rasters are used as inputs to estimate biogas potential from animal manure.

| File | Format | Description | Source |
|------|--------|-------------|--------|
| `Buffaloes/*.tif` | Raster (GeoTIFF) | Buffalo density (heads per km²) | FAO / GLW3 |
| `Cattle/*.tif` | Raster (GeoTIFF) | Cattle density (heads per km²) | FAO / GLW3 |
| `Chicken/*.tif` | Raster (GeoTIFF) | Chicken density (heads per km²) | FAO / GLW3 |
| `Goats/*.tif` | Raster (GeoTIFF) | Goat density (heads per km²) | FAO / GLW3 |
| `Pigs/*.tif` | Raster (GeoTIFF) | Pig density (heads per km²) | FAO / GLW3 |
| `Sheep/*.tif` | Raster (GeoTIFF) | Sheep density (heads per km²) | FAO / GLW3 |

### 3. Processed Data (`Processed_Data_Laos/`)

This folder is generated automatically by the `OnStove_notebook_Laos.ipynb` (`DataProcessor` steps). All layers are reprojected to **UTM Zone 48N (EPSG: 32648)** and spatially aligned to the population raster grid before being saved here.

| Sub-folder | Contents |
|------------|----------|
| `Base/` | Aligned base raster |
| `Biomass/Forest/` | Processed forest cover raster |
| `Biomass/Friction/` | Processed friction surface raster |
| `Demographics/population/` | Processed population raster |
| `Electricity/MV_lines/` | Reprojected MV power lines |
| `Electricity/Night_time_lights/` | Processed night-time lights raster |

## How to Use

1. **Clone this repository** and place all files according to the directory structure above.
2. **Create the conda environment** following the [OnStove installation guide](https://github.com/Open-Source-Spatial-Clean-Cooking-Tool/OnStove).
3. **Open** `OnStove_notebook_Laos.ipynb` in Jupyter.
4. **Run the DataProcessor cells** to read raw GIS layers, reproject, align, and save outputs to `Processed_Data_Laos/`.
5. **Run the OnStove model cells** to load processed data, read scenario specs (`soc_specs_lao.csv`, `tech_specs_lao.csv`), calibrate, and run the analysis.
